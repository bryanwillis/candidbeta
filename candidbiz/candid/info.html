    Text Logo Instructions

    Example: Text Logo

    Use the html for the logo in the Text Logo page. Enter your site name. Depending on your site's name, you'll need adjust the appropriate css in the base and in the media queries max-width 979 in the style.css file.

    SVG Logo Instructions

    This theme is retina ready. It uses an SVG image for the logo. It uses fonts to make the silhouetted icons. The colorful social icons are svg background images with png fall backs for legacy browsers.

    Logo maximum width is 405px IF you keep the links to the same amount of characters, pretty much. The max height is 65 px and you will need to adjust the top margin with the math provided in the css.

    1. Grab your vector logo and export as svg with the settings in this screen shot of settings
      • SVG 1.1
      • Link images (not embed)
      • No more than 2 Decimals
      • Presentation Attributes (a choice in Save for Web & Devices in Illustrator)
    2. Open the .svg in your code editor (NOT word or anything that will mess up the file).
    3. Remove the fixed size in the code SCREEN SHOT
    4. Add preserve aspect ratio at the end (watch out to keep the greater than) the code SCREEN SHOT 
      preserveAspectRatio="xMidYMid meet"
    5. Create a png, transparent background logo for the fallback
    6. Save the logos in the assets/images/ directory and update the html in the header to reflect the file name changes (if you didn't name the files logo.svg and logo.png).
    7. Adjust the #logo css in the style.css if your logo is of a different width and height (the logo in this theme is 255px X 55px) 
      #logo,
.logo-svg {
    height: 55px;
    width: 205px; 
}

      The mobile adjustments are not necessary.

  • Container/Scaffolding

    Bootstrap uses the class .container as the max-width that the content will not exceed. In this theme, the max-width is 1200px with some padding in percentages (it's in the top of the style.css file). To make it fluid and have zero horizontal bars, the .container class over-rides all .container classes in all media queries in the core bootstrap css.

    When setting up a page, think about what you want to be no more than 1200px wide and fluid and what you want to have a 100% as wide as the device/screen. For example the .strip class used on the home page is 100% width of the device so that is used OUTSIDE the .container.

    On the hero-unit page, the .hero-unit is inside a .strip and inside that strip is a .container. This is a common set up where the color or shading goes past the max-width.

    .container

    IF your entire page is contained within the max-width, AFTER the page-header, you'd write:

    <div class="container main-content">

Content goes here of the entire body of page IF the entire body of the page does not exceed the max width set on the .container class.

</div><!--close .container.main-content-->

    If only parts of your page are to be contained in the max-width, then you'd write something like this (after the page-header):

    
<div class="main-content">

<div class="strip"> 
<div class="container">
Content that is not to exceed the max width but inside a 100% div or section (like the .strip used on the home page).
</div><!--close .container-->
</div><!--close .strip-->

<div class="container">
Content that does not exceed the max width
</div><!--close .container-->

</div><!--close .main-content-->

    .span12

    .span12 is not necessary to use. Any content inside a .container doesn't exceed the max width set on the .container class.

    Nested Rows

    This theme is fluid, so the only class used outside .span2, .span3, .span4, .span5, .span6, .span7, .span8, .span9, .span10, and span11 is "row-fluid", you can put row-fluids inside row-fluids and create nested columns. The ONLY contents just inside a .row-fluid This is not a change from the standard bootstrap scaffolding, which you should learn more in detail at Boostrap's site.

    Remember span12 is not necessary and neither is row-fluid if you are using 100% of the width inside a .container. I often see themes with too many wrappers that do the same thing: wrapping content that needs only just one wrapper, usually .container. Since this theme is fluid, has no ugly scroll bars when AT any size, please look at my well comment code to follow along.

    For a good example of nested row-fluids, please check out the very well commented generic-layout-1 page.

  • Page Columns

    Pages can come in various column setups. These are the columns of the entire body of the page. I've built upon the Bootstrap columns but have done a little css so that the mark up stays the same and the order of content is SEMANTICAL. To elaborate: Bootstrap has rows and columns and if you want a fat column and a skinny column, say a primary and secondary column you'd create a row (row-fluid) and inside that the containers with classes .span9 and .span3 for a right sided side bar. That's easy enough and the primary content is first in the mark up, but if you wanted that side bar visually on the left, and the content visually on the right, you'd have to change the markup, and that changes the way it appears in mobile, with the least important content on top. The way I've set it up is to add a class, such as "sidebar-left" or "sidebar-right" to the "row-fluid" outer wrapper and classes "primary-column" to the .span9 and "secondary-column" to the .span3 respectively. So if you want the sidebar on the right or left, you just add the appropriate class to the parent row-fluid wrapper. The same thing goes for three columns, so you can have three columns like so: primary, secondary, tertiary, but have them appear as that or secondary, primary, tertiary (holy grail). Okeydokie.

    Sidebar on Left

    <div class="row-fluid sidebar-left"><!--note the "sidebar-left" class-->
	<!--begin row-fluid sidebar-left-->
		<div class="span9 primary-column">
		<!--begin div containing the primary content -->
Primary main content
		</div>
		<!--close div span9 primary-column-->
		<section class="span3 sidebar secondary-column">
Secondary sidebar content (widgets)
		</section>
		<!--close section secondary-column sidebar span3-->
	</div>
<!--close row-fluid-->

    Sidebar on Right

    <div class="row-fluid sidebar-right"><!--note the "sidebar-right" class-->
	<!--begin row-fluid sidebar-right-->
		<div class="span9 primary-column">
		<!--begin div containing the primary content -->
Primary main content
		</div>
		<!--close div span9 primary-column-->
		<section class="span3 sidebar secondary-column" id="secondary-nav">
Secondary sidebar content (widgets)
		</section>
		<!--close section secondary-column sidebar span3-->
	</div>
	<!--close row-fluid-->

  • FancyBox License

    Since this theme supports legacy browsers (up to a point), the choices for a responsive lightbox came to one, though more may be coming out as I write this. This was after a lot of testing of others. The choice is: fancyBox 2. This is the lightbox script that presents images, inline content, ajax, videos, and iframes in a "pop up" div. You can see examples throughout this site and below.

    I have permission to use my copy of fancyBox 2 in this theme.

    This theme contains "fancyBox" by fancyApps . Please note, that fancyBox is licensed under the terms of the Creative Commons Attribution-NonCommercial 3.0 License . If you would like to use fancyBox for commercial purposes, you can purchase a license from http://fancyapps.com/store/

    Order fancyBox today!

    I chose fancyBox 2 because it's responsive, it works on ie7+ and the latest browsers, it works on Android, IOS devices (iPad/iPhone), and it works in Opera Mobile. Plus, it handles more than just images. Other scripts are wonky on mobile or they are not ie7 friendly, or they have no license for theme development or one thing or another. FancyBox 2 is also the least expensive option as far as compatible scripts go, if you look at the features. The purchase price is $19.00 per commercial site, there's also a developer license.

    How to Use FancyBox with this Bootstrap Theme

    A single image:

    Add class "fancysingle" to the a tag, if you want a title, add the title to the tag as seen in the example above.

    <p><a class="fancysingle" href="../demo/1-lrg.jpg" title="Image Title"><img src="../demo/1-sm.jpg" alt="" /></a></p>

    Group of images with thumbnails and helper buttons:

    Add class "fancyme" to the a tag and create a data-fancybox-group of the same name for all images.

    <ul class="demo-gallery clearfix">
		<li>
			<a class="fancyme" data-fancybox-group="whatever" href="../demo/1-lrg.jpg" title="Image Title 1"><img src="../demo/1-sm.jpg" alt="" /></a>
		</li>
		<li>
			<a class="fancyme" data-fancybox-group="whatever" href="../demo/2-lrg.jpg" title="Image Title 2"><img src="../demo/2-sm.jpg" alt="" /></a>
		</li>
		<li>
			<a class="fancyme" data-fancybox-group="whatever" href="../demo/3-lrg.jpg" title="Image Title 3"><img src="../demo/3-sm.jpg" alt="" /></a>
		</li>
</ul>

    In this theme, I've combined all the fancybox scripts in one and compressed, the uncompressed version is also included. Calling all the functions available in these is done by putting the following in your footer before the closing body tag:

    <script src='assets/js/fancybox/source/custom-fancybox-combined.js' defer>></script><!--all fancy box buttons, media, helpers, thumbs AND loading-->

    ****If you were NOT using this theme, the script loading in the footer before the closing body tag would be:

    <script src="assets/js/fancybox/source/jquery.fancybox.pack.js"></script><!--main fancybox-->
<script src="assets/js/fancybox/source/helpers/jquery.fancybox-media.js"></script><!--fancybox media-->
<script src="assets/js/fancybox/source/helpers/jquery.fancybox-buttons.js"></script><!--fancybox buttons-->
<script src="assets/js/fancybox/source/helpers/jquery.fancybox-thumbs.js"></script><!--fancybox thumbs-->
<script src="assets/js/fancybox_init.js"></script><!--fancybox load-->

  • Twitter Feed?

    There is no twitter feed in this theme. You'll also notice that this is not a feature of many themes, but I really liked it. Twitter changed its API and now requires authentication. You will need to create an app with Twitter.

    All themes, mine or otherwise, with Twitter feeds will no longer work when they retire the API 1.0 and start using 1.1. This is not merely a change in the url, it requires OAuth.

    Options with instructions

    Sea of Clouds Modified with php to create a twitter feed, you can combine this with flexslider and make a sliding twitter feed

    Requires creating an app, php, getting secret keys, and so forth. Read the instructions and create an app account with twitter. It's a big production.

    Embedded timelines which you have to create with your own twitter account and domain for authentication

    API v1 retirement

    You can follow sea of clouds issue 264 and find out more about the twitter api and the lack of client side solutions once they shut down the old api: https://github.com/seaofclouds/tweet/issues/264
